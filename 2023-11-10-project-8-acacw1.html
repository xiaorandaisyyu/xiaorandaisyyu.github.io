Microarchitecture optimisation in splay trees (Advanced Computer Architecture CW1)
2023 November 10th

[Report PDF]

Overview

I studied the effect of various architectural features using the SimpleScalar simulator on the performance of the benchmark program, splay trees, and found the sweet spot architecture that ran this program with optimum energy efficiency, as part of my third year Advanced Computer Architecture coursework. The coursework was completed individually, and I received high marks (A*, 17/20). The evaluation included a bash script that runs the simulation of the best configuration and a written report on my systematic strategy to find the optimum implementation and analysis of the experiment results.

Project Outline & Personal Work

I observed the functioning and effects of various components of the computer architecture, each parameter individually and sometimes several together to study how they interact and become each other’s bottleneck, in order to find the best trade-off overall. Apart from total energy consumption, I also investigated into their effects on the program’s running speed (instructions completed per clock cycle, start-to-finish time), and reached the conclusion that race to finish doesn’t always win (power consumption-wise).

Components/parameters tested include: Register Updata Unit and Load-Store Queue size, L1 & L2 cache configurations (number of sets, block size, number of ways/associativity, cacheline replacement policy, hit latency), Branch Predictor configurations(predictor type, Branch Target Buffer configurations, Return Address Stack configurations), Translation Lookaside Buffer (for both instruction and data) configurations, instruction decode/issue/commit bandwidth, number of ALUs.
